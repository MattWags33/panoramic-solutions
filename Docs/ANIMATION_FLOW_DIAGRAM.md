## The Magic of `await setTimeout(0)`

### What It Does

```javascript
// Set flags
setIsPreparingAnimation(true);
setIsAnimatingGuidedRankings(true);

// WITHOUT this line: React might not process flags yet
// WITH this line: React MUST process flags before continuing
await new Promise(resolve => setTimeout(resolve, 0));

// Now it's safe to continue - flags are guaranteed updated
```

### Why It Works

1. **JavaScript Event Loop**: `setTimeout(0)` schedules callback for next event loop iteration
2. **React Batching**: React flushes all pending state updates before next iteration
3. **Await**: Function pauses until callback executes
4. **Result**: All state updates are processed before continuing

### Performance

- **Overhead**: ~1-5ms per call
- **Total Added**: ~2-10ms for 2 calls
- **Impact**: Negligible (user won't notice)
